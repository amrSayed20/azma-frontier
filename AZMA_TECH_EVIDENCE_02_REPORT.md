# AZMA Technology Evidence Collection 02 Report

## Executive Summary

Technology Evidence Collection 02 for the Image Generation Ecosystem is complete as an evidence-only constitutional activity.

Created:
- AZMA_TECH_EVIDENCE_02_IMAGE.md
- AZMA_TECH_EVIDENCE_02_REPORT.md

Evidence was collected from official documentation and policy sources for six major image-generation families: OpenAI GPT Image, Google Gemini/Nano Banana plus Imagen surfaces, Stability Stable Image, Adobe Firefly, Black Forest Labs FLUX, and Midjourney.

Coverage was recorded across all required categories: model architecture, licensing, image generation, image editing, inpainting/outpainting support, control mechanisms, character consistency, fine-tuning, self-hosting, API availability, enterprise readiness, community maturity, documentation quality, release cadence, maintenance activity, security considerations, and operational considerations.

Evidence highlights (facts only):
- Image generation and image editing capabilities are explicitly documented across all six families.
- Inpainting support is explicitly documented for OpenAI and Google; Stability explicitly documents mask-based inpainting; Midjourney documents inpainting/outpainting tools in version guidance; FLUX explicitly documents outpainting tools.
- API availability is explicit for OpenAI, Google, Stability, Adobe, and BFL. Midjourney public API availability was not established from retrieved official pages and is marked Unknown.
- Fine-tuning/customization evidence is explicit for Adobe (Custom Models API) and BFL (fine-tuning/LoRA licensing rights). OpenAI gpt-image-2 explicitly lists fine-tuning as not supported.
- Self-hosting evidence is explicit for BFL commercial weight licensing and Stability enterprise on-prem/self-hosted options; absent/unknown in retrieved official pages for OpenAI, Google, Adobe, and Midjourney.
- Enterprise and security evidence is strongest where explicit compliance/trust references were present (for example OpenAI trust/compliance pages; BFL trust/security links; Stability enterprise/SLA references).

Unknowns were explicitly marked wherever official-source confirmation was not retrieved, including architecture internals for several families and selected enterprise/security details for some providers.

No recommendation, ranking, benchmark conclusion, implementation action, or architecture decision was performed.
